## URL pattern notes (technical)

- **`.html` vs extensionless:** Blog posts and some tools use `.html`; product URLs use **trailing slashes**. Mixed patterns increase risk of **duplicate signals** if both resolve without a single canonical; verify redirects/canonicals for every template.
- **`/blog/how-to-add-moringa-to-diet` vs `…/how-to-add-moringa-to-diet.html`:** The live page metadata uses the `.html` URL as `og:url` — any bare slug without `.html` should 301 to the canonical or return 404.
- **Uppercase / underscores:** Sitemap and sampled URLs are **lowercase, hyphenated**; no material uppercase-in-path issues in the sample.
- **Query strings:** No heavy `?id=` / `&ref=` patterns observed on sampled marketing URLs.

## Gaps and risks

1. **Sitemap smaller than the crawlable site:** Several live articles appear in maps or internal search but are **not** in `sitemap.xml` — **incomplete sitemap** limits predictable indexing.  
2. **Canonical and duplicate paths:** Enforce a single URL per post (`.html` canonical is strong on `how-to-add` sample).  
3. **Orphan / weak-link risk:** Low-link utility pages (e.g. some Labs tools) need links from the hub (`/nutrithrive_labs/`) and footer.
